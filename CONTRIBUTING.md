# Contributing to Spotify Skip Tracker

First of all, thank you for your interest in contributing! 🎉

Whether you've found a bug, have an idea for a new feature, or want to improve the codebase, all contributions are welcome.

## Before you start

Please:

- Check existing Issues before creating a new one.
- Open an Issue first for larger changes so we can discuss the approach.
- Keep pull requests focused on a single feature or bug fix.

## Reporting bugs

When reporting a bug, please include:

- Operating system
- Browser (if applicable)
- Steps to reproduce the issue
- Expected behaviour
- Actual behaviour
- Screenshots or logs if available

## Suggesting features

Feature requests are welcome.

Please describe:

- The problem you're trying to solve
- Your proposed solution
- Any alternative ideas you've considered

## Development

### Backend

- Python
- Flask
- PostgreSQL (Neon)
- Railway

### Frontend

- React
- TypeScript
- Vite

### Before submitting a Pull Request

Please make sure that:

- Code follows the existing style.
- New functionality includes tests where appropriate.
- Existing tests pass.
- There are no unnecessary formatting-only changes.

## Pull Requests

Please keep pull requests:

- Small and focused
- Well described
- Linked to an Issue whenever possible

## Security

If you discover a security vulnerability, please do **not** open a public issue.

Instead, contact the maintainer privately.

## Questions

If you're unsure about anything, feel free to open a Discussion or an Issue before starting work.

Thanks for helping make Spotify Skip Tracker better!
